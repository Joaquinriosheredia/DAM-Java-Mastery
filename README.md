# 🚀 DAM-Java-Mastery: Portfolio Técnico Avanzado

Este repositorio es un diario técnico de aprendizaje y un escaparate de soluciones avanzadas desarrolladas durante el Ciclo Superior de **Desarrollo de Aplicaciones Multiplataforma (DAM)**. 

El objetivo es demostrar competencia técnica en la arquitectura de software, concurrencia, persistencia de datos y desarrollo móvil, manteniendo una racha de contribuciones constante y valiosa.

---

## 🛠️ Stack Tecnológico

| Área | Tecnologías |
| :--- | :--- |
| **Lenguajes** | Java 17+, SQL, PL/SQL, XML, JSON, R, Bash. |
| **Frameworks/Librerías** | Hibernate (JPA), JavaFX, JUnit 5, Mockito. |
| **Bases de Datos** | PostgreSQL, MySQL, SQLite, MongoDB (NoSQL). |
| **Entornos/Herramientas** | Android Studio, IntelliJ IDEA, Scene Builder, Git. |
| **Infraestructura** | Linux (WSL2), Docker, Ollama (Local AI). |

---

## 📂 Estructura del Proyecto

El repositorio está organizado jerárquicamente siguiendo las asignaturas clave del currículo de ingeniería de software:

* **[/PSP](./PSP):** Programación de Servicios y Procesos. Enfoque en multihilo (Threads), sincronización de recursos y sockets TCP/IP.
* **[/BBDD_Acceso](./BBDD_Acceso):** Gestión de persistencia. Incluye scripts SQL complejos, procedimientos almacenados y mapeo ORM con Hibernate.
* **[/Android_PMDM](./Android_PMDM):** Desarrollo de aplicaciones móviles. Ciclo de vida de actividades, layouts y persistencia SQLite.
* **[/Interfaces_DI](./Interfaces_DI):** Diseño de interfaces bajo arquitectura MVC con JavaFX y estándares de usabilidad (UX).
* **[/BigData_LMSGI](./BigData_LMSGI):** Análisis de grandes volúmenes de datos, lenguajes de marcas (XML/JSON) y transformaciones XSLT.
* **[/Core_Prog](./Core_Prog):** Fundamentos de POO, algoritmos, estructuras de datos y patrones de diseño (Singleton, Factory, DAO).

---

## 🤖 Automatización y Calidad

Este portafolio utiliza un sistema de automatización basado en **IA Local (Qwen 2.5 vía Ollama)** ejecutado en un procesador **Ryzen 7**. 

* **Daily Streak:** Generación automática de retos técnicos diarios.
* **Conventional Commits:** Mensajes de commit estandarizados (`feat:`, `refactor:`, `docs:`) para un historial limpio y profesional.
* **Clean Code:** Todas las soluciones siguen principios SOLID y Javadoc para garantizar la legibilidad.
